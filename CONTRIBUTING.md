# Contributing Guidelines

## Structure

Follow the existing directory structure:
- Use markdown (.md) for all documentation
- Name files descriptively: `docker-networking.md` not `docker.md`
- Include a README.md in each subdirectory

## Format

- Use heading hierarchy: H1 for title, H2 for sections
- Add table of contents for longer documents
- Include practical examples where possible
- Link to related documents using markdown links

## Templates

Use provided templates in `templates/` directory as starting point.

## Quality Standards

- Proofread before committing
- Test all code examples
- Keep examples up-to-date with current versions
- Cross-reference related topics

## Process

1. Create a feature branch
2. Add/update documentation
3. Validate internal links
4. Submit pull request
